# PR #135970 — fix(endpointslice): ensure empty endpoints not serialized as null

## Issue: #135968
EndpointSlice with 0 endpoints serialized as `endpoints: null` instead of `endpoints: []`, breaking strict JSON clients.

## Ground Truth
- In `staging/src/k8s.io/endpointslice/reconciler.go`: add `&& slicesToDelete[0].Endpoints != nil` to the placeholder comparison condition
- Remove the now-inaccurate comment about "unnecessarily delete/recreate"
- Effect: if existing placeholder has nil Endpoints, skip the optimization → falls through to delete+create with proper `Endpoints: []`
- New standalone test `TestReconcileFixesNilEndpoints` in reconciler_test.go
- Size: +59/-2 across 2 files (S)

### Key Observations
- **Three different fix architectures**: RAG and Hybrid both modified the callsite (if/else branches), Local modified the comparator itself. GT was simplest — just appended `&& Endpoints != nil` to the condition.
- **None found the minimal GT fix**: All produced more complex solutions. The GT's elegance is that it just lets the existing delete+create path handle it naturally by not short-circuiting.
- **Hybrid had the best overall solution**: update-in-place is actually more efficient than GT's delete+create approach (1 API call vs 2), with excellent tests.
- **Local's comparator fix is interesting**: fixing the root cause (comparison doesn't distinguish nil vs empty) is arguably more correct, but results in extra API churn.
- **First PR where Hybrid clearly won**: the combination of RAG discovery + local file reading produced the best-tested, most efficient fix.
